    # UX Copy Patterns

    This section showcases UX microcopy patterns designed using modern UI writing principles inspired by enterprise SaaS best practices.

    The focus is on clarity, brevity, and confidence—helping users complete tasks without friction.

    Design principles followed

    • Clear and concise language
    • Action-oriented primary buttons
    • Neutral, non-blaming tone
    • Progressive disclosure for advanced actions
    • Consistent terminology across features

    Modal windows

    Lock Forecast Version

    Title: Lock forecast version
    Body: Locking this forecast prevents further edits. You can still view and report on the data.
    Primary CTA: Lock forecast
    Secondary CTA: Cancel

    Submit Budget for Approval

    Title: Submit budget for approval
    Body: After submission, changes will require approval. Make sure all assumptions are final.
    Primary CTA: Submit
    Secondary CTA: Review again

    Delete Forecast Version

    Title: Delete forecast version
    Body: This action permanently removes the forecast and cannot be undone.
    Primary CTA: Delete
    Secondary CTA: Cancel

    Confirmation messages

    Forecast locked

    Message: Forecast version locked.
    Supporting text: Editing is now disabled for this version.

    Budget submitted

    Message: Budget submitted for approval.
    Supporting text: You’ll be notified once a decision is made.

    Snackbar notifications

    Successful save

    Message: Changes saved
    Action (optional): Undo

    Forecast recalculated

    Message: Forecast updated using latest assumptions

    AI insights available

    Message: New AI insights are available
    Action: Review

    Informational messages

    AI insights disclaimer

    Text: AI insights are based on historical patterns and do not change your data automatically.

    Forecast version status

    Text: This forecast is currently locked and available in read-only mode.

    Warning states

    Large variance detected

    Title: Review recommended
    Message: Significant variance detected compared to historical trends.

    Missing required drivers

    Message: Some required drivers are missing. Complete setup to continue.

    Validation errors

    Invalid growth rate

    Message: Enter a value between 0% and 100%.

    Required field missing

    Message: This field is required.

    Duplicate version name

    Message: A forecast version with this name already exists.

    Toggle controls

    Enable AI insights

    Label: AI-assisted insights
    Helper text: Highlights trends and anomalies using historical data.

    Lock after submission

    Label: Lock forecast after submission
    Helper text: Prevents further edits once submitted for approval.

    Inline help text

    Forecast assumptions

    Text: Assumptions are applied across the selected time period unless overridden.

    Approval workflow

    Text: Approval workflows ensure forecasts meet governance requirements before finalization.

    Empty states

    No AI insights available

    Title: No insights yet
    Body: Add historical data or adjust assumptions to generate insights.

    No forecast versions

    Title: No forecasts created
    Body: Create a forecast to start planning and analysis.

    Error states

    Calculation failed

    Title: Something went wrong
    Body: We couldn’t complete the calculation. Try again or contact support if the issue persists.

    Success states

    Forecast created

    Message: Forecast created successfully

    Budget approved

    Message: Budget approved and locked for reporting
